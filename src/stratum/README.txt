------ Объяснение структуры каталогов и зависимостей модулей. ------

Часть 1: Набор хелперов. Слабо зависит от платформы.

 - helpers - независимый каталог. Функции отсюда могут быть вполне использованы в
других проектах.
 - fileFormats - зав. от helpers. Описания форматов файлов, функции их чтения.
 - common - зав. от helpers,fileFormats. Общие типы системы Stratum.

Часть 2: Графическая подсистема. Сильно зависит от DOM API (SVG, Canvas, CSS и т.п.).

 - graphics - реализация графики, слабо зависит от common,helpers (из-за декодировки значений COLORREF и кодов клавиш).
По идее графику можно вынести в отдельный проект и сделать библиотеку общего назначения.

Часть 3: Компоненты системы Stratum. Схема имиджей.

 - compiler - компилятор кода имиджей, зависит от common,fileFormats,helpers.
 - classLibrary - библиотека прототипов имиджей, зависит от вышеперечисленного.
 - project - зависит от вышеперечисленного и API stratum.d.ts (в плане использования файловой системы).
 project состоит из двух частей: 1) Вычисляемый узел схемы имиджей; 2) Проект, содержащий ссылку на корневой узел и общую память всей схемы.

--------------
Как можно заметить, схема и графика никак между собой не связаны.
Однако, в Stratum есть функции типа RegisterObject, когда по нажатии на объект должен выполниться узел схемы.
Поэтому в качестве "прокси", который связывает их между собой выступает enviroment, описаный далее.
--------------

 Часть 4:
  - enviroment - менеджер, который связывает все воедино. Хранит состояние открытых проектов, окон, пространств и т.п.
